API Para Cadastro de Clientes e Ordens de Serviços


Esta página inicial foi criada para indicar ao usuário os principais endpoints desta API.

A API funciona seguindo o seguinte relacionamento entre as entidades:
  • Um cliente pode possuir várias ordens de serviço.
  • Uma ordem de serviço precisa necessariamente estar vinculada a um cliente.
  • Uma ordem de serviço pode estar com os seguintes status: ABERTA, CANCELADA ou FINALIZADO
  • Uma ordem de serviço pode ter 0 ou mais comentários.
Em caso de dúvidas, acesse a documentação da API no ink abaixo:
Documentação da API

EndPoints:

Para a entidade "Clientes"

  • "https://api-client-serviceorder.herokuapp.com/clientes" (usando o método GET) para listar todos os clientes cadastrados.
  • "https://api-client-serviceorder.herokuapp.com/clientes/{id_cliente}" (usando o método GET) para trazer o cliente cadastrado com o id(número inteiro) informado.
  • "https://api-client-serviceorder.herokuapp.com/clientes" (usando o método POST) para criar um cliente com as informações solicitadas, sendo elas: Nome, E-Mail e Telefone (todos do tipo String).
  • "https://api-client-serviceorder.herokuapp.com/clientes/{id_cliente}" (usando o método POST) para atualizar os dados de um cliente com as informações passadas. Lembrando que as informações possíveis de serem passadas ao servidor são: Nome, E-Mail e Telefone (todos do tipo String).
  • "https://api-client-serviceorder.herokuapp.com/clientes/{id_cliente}" (usando o método DELETE) para deletar o cliente cadastrado com o id(número inteiro) informado.

Para a entidade "Ordem de Serviço"

  • "https://api-client-serviceorder.herokuapp.com/ordemservico" (usando o método GET) para listar todas as ordens de serviço cadastradas.
  • "https://api-client-serviceorder.herokuapp.com/ordemservico/{id_ordemservico}" (usando o método GET) para trazer a ordem de serviço cadastrada com o id(número inteiro) informado.
  • "https://api-client-serviceorder.herokuapp.com/ordemservico" (usando o método POST) para criar uma ordem se derviço com as informações solicitadas, sendo elas: Descrição (String), Preço (double) e Id do CLiente (número inteiro).
  • "https://api-client-serviceorder.herokuapp.com/ordemservico/{id_ordemservico}" (usando o método PUT) para atualizar os dados de uma ordem de serviço. Lembrando que as informações possíveis de serem passadas ao servidor são: escrição (String), Preço (double) e Id do CLiente (número inteiro).
  • "https://api-client-serviceorder.herokuapp.com/ordemservico/{id_ordemservico}" (usando o método DELETE) para deletar a ordem de serviço cadastrada com o id(número inteiro) informado.
  • "https://api-client-serviceorder.herokuapp.com/ordemservico/{id_ordemservico}/cancelar" (usando o método PUT) para alterar o status da ordem de serviço cadastrada com o id(número inteiro) informado para CANCELADO.
  • "https://api-client-serviceorder.herokuapp.com/ordemservico/{id_ordemservico}/finalizar" (usando o método PUT) para alterar o status da ordem de serviço cadastrada com o id(número inteiro) informado para FINALIZADO.

Para a entidade "Comentário"

  • "https://api-client-serviceorder.herokuapp.com/ordemservico/{id_ordemservico}/comentario" (usando o método GET) para listar todos os comentários de uma ordens de serviço cadastrada.
